# 出力フォーマット仕様

Phase 4 (SYNTHESIZE) で最終出力を生成する際にこの仕様に従う。
5つのファイルに分割して出力する。

## 参照表記ルール（全ファイル共通）

すべてのファイルで以下の表記を統一する。表記揺れは許容しない。

| 対象 | 形式 | 例 |
|---|---|---|
| 海外資料 | `資料/海外NN: タイトル(著者)` | 資料/海外06: ビルドトラップ(Perri) |
| 日本資料 | `資料/日本NN: タイトル(著者)` | 資料/日本03: プライシング戦略(高橋) |
| 事例 | `事例NN: 企業名` | 事例01: Spotify |
| 論点 | `論点N: 内容の要約` | 論点3: JTBDの二大学派 |
| サブトピック | `サブトピック: 名称` | サブトピック: ユニットエコノミクス |

**禁止表現:**
- 「あなた」「あなたの」は使用しない
- 代わりに「今回の文脈」「依頼の背景」「対象の状況」等の客観的表現を使う

**引用資料の記載位置:**
- テーブルのセル内で資料を引用する場合、引用は**セルの末尾**に記載する
- 例: `価格弾力性を重視すべき（資料/海外02: Monetizing Innovation(Ramanujam)）`

## 出力ファイル構成

```
$SESSION_DIR/
├── [トピック]_[YYYYMMDD].md  ← ブリーフィング（①〜④のサマリー、単体シェア用）
├── 01_structure.md           ← ① テーマの構造と全体像
├── 02_references.md          ← ② 参考資料と要約
├── 03_cases.md               ← ③ 代表事例
├── 04_insights.md            ← ④ 学びと示唆
└── .intermediate/
```

ブリーフィングのファイル名例: `ファイナンス_20260330.md`, `RAG_20260331.md`

生成順序（依存関係を考慮）:
1. 01_structure.md
2. 02_references.md
3. 03_cases.md（①②の内容を踏まえて事例を選定）
4. 04_insights.md（③の事例も踏まえて示唆を生成）
5. [トピック]_[YYYYMMDD].md（①〜④のサマリーとして最後に生成）

## ブリーフィング [トピック]_[YYYYMMDD].md

①〜④のエッセンスを凝縮した概要。これ1枚で全貌を把握でき、単体で他者にシェアできる。
5-10分で読める分量を目安とする。

```markdown
# [トピック] ブリーフィング
> 生成日: YYYY-MM-DD
> 文脈: [Phase 0で確定した文脈]
> 調査深度: クイック / スタンダード / ディープ
>
> 詳細: [01_structure.md](./01_structure.md) | [02_references.md](./02_references.md) | [03_cases.md](./03_cases.md) | [04_insights.md](./04_insights.md)

---

## テーマ概要
（①の1.1〜1.2を圧縮。3-5文でこのテーマが何で、なぜ重要かを説明）

## 主要な論点
（①の1.3を圧縮。論点テーブルから最重要3-5点を抜粋）

## 全体の概念構造
（①の1.6 サブトピックの関係性をそのまま記載。
 主要なサブトピックとその関係を構造的に記述。テキストまたは簡易図で表現）

## 場面別の要注意ポイント
（①の1.7を圧縮。依頼の背景に最も近い2-3場面のみ抜粋）

## 注目事例
（③から最も参考になる1-2事例を抜粋。企業名/施策概要/教訓を各2-3文で）

## 重要資料 Top 5
（②のランキングから上位5件を抜粋。タイトル/著者/一言/URL のみ）

## 主要な学び
（④の全体的な学びから最重要3-5点を抜粋）

## 今回の文脈への示唆
（④の文脈への示唆を1段落に圧縮。推奨する最初の一歩を1文で）
```

## ① 01_structure.md（テーマの構造と全体像）

テーマの全体像を深く理解するためのファイル。
全体像・用語集に加え、資料横断の深堀りとサマリーを含む。

```markdown

---

（3-5文。専門用語を避けた平易な説明）

### 1.2 なぜ今重要か
（2-3文。時事性・社会的背景）

### 1.3 主要な論点と対立構造

| # | 論点 | 立場A | 立場B | 備考 |
|---|---|---|---|---|
| 1 | 論点1: 〇〇 | 1-2文 | 1-2文 | ... |
| 2 | 論点2: △△ | 1-2文 | 1-2文 | ... |

### 1.4 歴史・経緯

| 年代 | 出来事 | 意義 |
|---|---|---|
| ... | 1-2文 | 1文 |

### 1.5 現在のトレンドと今後の方向性
（箇条書き5-10点）

### 1.6 サブトピックの関係性
（主要なサブトピックとその関係を構造的に記述。テキストまたは簡易図で表現。
 各サブトピックは「サブトピック: 名称」の形式で表記）

### 1.7 場面別の考慮事項マップ

**Step 1: 観点による状況の把握**

| 観点 | 選択肢・分岐 |
|---|---|

**Step 2: 場面ごとの主要論点**

| 場面（観点の組み合わせ） | 主要論点 | 特に気をつけるべきこと |
|---|---|---|

各場面の論点は、参照表記ルールに従って紐づける
（例:「→ 論点3: JTBDの二大学派、資料/海外05: Competing Against Luck(Christensen)が詳しい」）。

## 2. 関連用語集

|---|---|---|---|

## 3. 資料横断の深堀り

複数の資料を横断して見えてくるテーマの深層構造。
個別資料のサマリー（02_references.md）とは異なり、資料間の関係を分析する。

### 3.1 資料群が共通して指摘するコアメッセージ
（複数の資料が共通して主張する核心的なポイント。箇条書き3-5点。
 各ポイントに「→ 資料/海外XX, 資料/日本XX」と参照を付ける）

### 3.2 資料間の見解の相違

| 論点 | 立場A | 立場B | 背景の違い |
|---|---|---|---|
| 論点N: 〇〇 | 主張内容（資料/海外XX: タイトル(著者)） | 主張内容（資料/日本XX: タイトル(著者)） | 背景説明 |

※ 引用資料はセル内の末尾に記載する。

### 3.3 時代・地域による視点の違い
（海外と日本の視点差、古い資料と新しい資料の変遷など）

### 3.4 統合サマリー
（このテーマの「現時点での最善の理解」を5-10文で記述。
 単一の資料では得られない、横断的に読んで初めて見えてくる全体像）
```

## 資料種別ごとのサマリーフォーマット

## 分量・粒度の制約

| セクション | 制約 |
|---|---|
| ブリーフィング全体 | 5-10分で読める分量 |
| ① 全体像 1.1 | 3-5文、専門用語を避ける |
| ① 論点テーブル | 1論点あたり各セル1-2文 |
| ① 歴史テーブル | 1行あたり「出来事」1-2文 |
| ① 用語集「定義」 | 2文以内 |
| ① 統合サマリー | 5-10文 |
| ② ストーリーライン（書籍） | 5-10文 |
| ② ストーリーライン（論文・記事） | 3-5文 |
| ② 章ごとサマリー | 3-5文/章 |
| ② 資料サマリー「一言」 | 1文（日本語50文字以内 / 英語16 words以内） |
| ② キーワード | 5個 |
| ④ テーマの本質 | 3-5文 |

## フォールバック

書籍の章構成やWeb記事の全文がWebFetch/WebSearchで取得できない場合:

1. 出版社・著者の公式ページから目次・概要を取得
2. 書評・要約サイト（Amazon レビュー、flier、ブクログ等）から章構成を推定
3. 複数の書評・紹介記事を横断的に参照してサマリーを構成
4. 上記すべて不十分な場合: 「情報不足」と明示し、取得可能な範囲での概要を記載

## ファイル間の相互参照ルール

- 各ファイルのヘッダーに他ファイルへの相対リンクを配置
- 資料・事例・論点の参照は冒頭の「参照表記ルール」に従う
- セクション番号はファイル内で独立（①の1.3、④の2.1 等）
