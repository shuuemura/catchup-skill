# 資料サマリーの品質要件

Phase 3 のバリデーション（Step 3-V2）で使用する。
各資料エントリが満たすべき最低基準と、A評価（優良）の基準を定義する。

## 1. 必須セクションの存在（全種別共通）

| セクション | 書籍 | 論文 | 記事/レポート |
|---|---|---|---|
| メタデータ（種別/URL/おすすめ度/一言/キーワード/全体像との対応） | 必須 | 必須 | 必須 |
| ストーリーライン | 必須 | 必須 | 必須 |
| 章ごとのサマリー | 必須 | 不要 | 不要 |
| 主要な主張・フレームワーク | 必須 | 必須（品質評価と併存） | 必須 |
| 品質評価 | 不要 | 必須 | 不要 |

セクション見出しが1つでも欠けている場合は **不合格** とする。

## 2. 定量的な最低基準（B評価の下限）

| 項目 | 最低基準 | 根拠 |
|---|---|---|
| ストーリーライン文字数 | 300字以上 | 300字未満のエントリは文脈説明が不足し「何の本/記事か」が伝わらない |
| ストーリーライン文数 | 3文以上 | 2文以下では問題提起→展開→結論の構造を表現できない |
| 主要な主張・フレームワーク箇条数 | 3点以上 | 資料の性質に応じて増減可。ただし3点未満では核心的主張をカバーできない |
| 章ごとのサマリー行数（書籍のみ） | 3行以上 | 3行未満では書籍の構造を把握できない |
| 章ごとのサマリー 1行あたりの要約文字数 | 60字以上（平均） | 60字未満では章の内容が伝わらない |
| 一言の文字数 | 50字以内 | フォーマット仕様による上限 |
| キーワード数 | 5個 | フォーマット仕様による |

## 3. 質的な最低基準（B評価の下限）

以下の3つの質的基準（具体性・メッセージ性・独自性）を満たすこと。

---

### 基準A: 具体性 — 抽象的な項目名ではなく、固有名詞・数値・構造が含まれているか

ストーリーラインと主要な主張の両方に適用する。

**チェックポイント:**
- [ ] フレームワーク名・モデル名が登場する（例: 「12ビルディングブロック」「Teach-Tailor-Take Control」）
- [ ] 数値が登場する（例: 「クォータ達成率+27.6%」「スターパフォーマーの40%」「6,000人調査」）
- [ ] 企業名・人名・出典名が登場する（例: 「Salesforceでの$500M→$3B成長」「CSO Insightsの調査」）
- [ ] 因果関係や条件が明示されている（例: 「〜の場合に〜が向上する」「〜なしには〜は機能しない」）

上記4つのうち **2つ以上** を満たすこと。

**NG:**
> テレワーク時代にオンライン商談が激増し、SEを提示する。山下貴宏氏が監修し、5フェーズを紹介。

→ 「SE」「5フェーズ」とあるが固有名や中身がなく、何の5フェーズかわからない。

**OK:**
> 日本企業の多くが営業人材育成を現場任せにし、一部の優秀者頼りになっている構造的問題から出発。「成果起点の育成フレームワーク」を提示し、データ整備→専任人材配置→プログラム開発→KPI設計→成果検証の5段階実装ステップを企業規模別に解説。Sansan・NTTコム・Salesforceの国内事例を通じて理論の実践的応用を示す。

→ フレームワーク名（成果起点の育成フレームワーク）、ステップの中身、企業名3社が入っている。

---

### 基準B: メッセージ性 — 体言止めの項目名ではなく、「何がどうだ」という主張になっているか

主に「主要な主張・フレームワーク」セクションに適用する。
各箇条書きが「名称の提示」で終わるのではなく、**その主張の中身・意味・帰結まで伝える文** であること。

**チェックポイント:**
- [ ] 各箇条書きが「名称: 説明文」の構造を持ち、説明文が主張の内容を読むだけで理解できる
- [ ] 体言止め（名詞で終わる）の箇条書きが全体の半数を超えていない

**NG（体言止めの羅列）:**
> - 5フェーズモデル
> - 仕組みとしてのSE
> - データ基盤の重要性

→ 項目名の羅列。「で、何なの？」が伝わらない。目次と変わらない。

**NG（説明があるが当事者でないと意味不明）:**
> - **3P整合**: People/Processes/Priorities

→ 略語の展開だけ。3つを整合させると何が起きるのか、なぜ重要なのかが書かれていない。

**OK（主張の中身まで伝わる）:**
> - **3P整合フレームワーク**: People（人材・文化）・Processes（プロセス・仕組み）・Priorities（戦略的優先事項）の三層を整合させることがイネーブルメント成功の必要条件
> - **ピアツーピア学習の制度化**: トップセールスの知識・スキルを組織に蓄積・共有するピアラーニング文化は、外部トレーニング以上に現場定着効果が高い

→ 名称 + 構造の説明 + 「なぜ重要か」「何が起きるか」まで含んでいる。

---

### 基準C: 独自性 — その資料ならではの知見が含まれ、一般論に終わっていないか

ストーリーラインと主要な主張の両方に適用する。
「どの資料にも書いてありそうなこと」だけで構成されていないこと。

**チェックポイント:**
- [ ] その資料固有の発見・フレームワーク・データ・視点が1つ以上含まれている
- [ ] 「営業は大事」「データ活用が重要」「組織的な取り組みが必要」のような一般論だけで終わっていない
- [ ] 他の資料との差別化ポイント（この資料を読む理由）が伝わる

**NG（一般論の集合）:**
> - 営業プロセスの可視化が重要
> - データに基づく意思決定をすべき
> - 組織的な育成体制を構築する

→ すべて正しいが、この資料「ならでは」の知見がゼロ。任意のSE資料に書ける。

**OK（その資料固有の知見）:**
> - **営業プロセス7分解**: 営業活動を7段階に分割して可視化し、どの段階にボトルネックがあるかを特定してピンポイントで改善
> - **82%は買わないという前提**: 営業の出発点を「いかに売るか」ではなく「なぜ買われないか」に置き換える発想転換

→ 「7分解」「82%は買わない」はこの資料（セールス・イズ）固有の概念。他の資料では出てこない。

**OK（視点の独自性）:**
> - **マーケター起点のSE:** 「SEはマーケターがオウンすべき機能」という立場から、コンテンツ・メッセージ・テクノロジーをマーケティング視点で設計するアプローチを提唱

→ 多くのSE資料が営業視点であるのに対し、マーケター視点というこの資料（Effective SE）固有のポジションが明確。

---

### ストーリーラインに含めるべき要素（まとめ）

- [ ] 資料の出発点（どんな問題意識・背景から書かれたか）
- [ ] 核心的な主張または発見（この資料が最も伝えたいこと）
- [ ] 展開の概要（どのように論を進めるか）
- [ ] 上記「具体性」「独自性」の基準を満たしている

### 主要な主張に含めるべき要素（まとめ）

- [ ] 各箇条が「名称: 主張の中身」構造を持つ（メッセージ性）
- [ ] 上記「具体性」「独自性」の基準を満たしている
- [ ] 読むだけで「この資料の核心」が伝わり、他の資料と区別できる

## 4. A評価の基準（優良）

最低基準に加え、以下を満たすエントリをA評価とする:

| 項目 | A評価の基準 |
|---|---|
| ストーリーライン | 400字以上かつ4文以上。「問題提起→著者の資格→核心的主張→展開方法→到達点」の5層構造 |
| 主要な主張 | 5点以上。すべての箇条が「名称: 主張の中身」構造を持ち、具体性・メッセージ性・独自性の3基準をすべて満たす |
| 章ごとのサマリー（書籍） | 4行以上。各行の要約が章の核心的主張を明示し、読むだけで章の内容が把握できる |
| エビデンス | ストーリーラインまたは主要な主張に、研究データ・実績数値・具体的事例が2つ以上含まれる |

### A評価の具体例

**海外08: The Challenger Sale（ストーリーライン）**

> CEB（Corporate Executive Board）が6,000人以上の営業担当者を対象に実施した大規模調査から始まり、ハイパフォーマーが5つのプロファイルのうち「チャレンジャー型」に集中していることを発見する。従来の常識だった「リレーションシップビルダー型」が複雑なB2Bセールスでは最も成果が低く（スターパフォーマーの7%のみ）、一方でチャレンジャー型はスターパフォーマーの約40%を占め、平均より15%高い成果を出すことが示される。チャレンジャーが持つ3つのコアスキルは「Teach（独自の洞察を教える）」「Tailor（メッセージを個別最適化する）」「Take Control（購買プロセスをコントロールする）」であり、これが「Teach-Tailor-Take Control」フレームワークとして体系化される。本書はSE領域において「何を標準化すべきか」を示した最も影響力の大きいフレームワーク書籍の一つとなった。

→ 研究規模（6,000人）、具体的数値（40%、7%、15%）、フレームワーク名（Teach-Tailor-Take Control）、歴史的位置づけ（SE領域で最も影響力）がすべて含まれている。

## 5. バリデーション手順

Step 3-V2 では以下の手順で品質チェックを行う:

1. 各バッチファイルを読み込む
2. 各資料エントリについて、セクション1の必須セクション存在を確認
3. セクション2の定量基準を確認（文字数・文数・箇条数）
4. セクション3の質的基準を確認（NG例に該当しないか）
5. 不合格項目がある資料を「補修対象リスト」として出力する
6. 補修対象リストにはエントリID・不合格項目・具体的な不足内容を記載する
