# /catchup -- 概念キャッチアップ Skill

新しいテーマの全体像を短時間で把握するためのSkill。
Subagentによる並列探索で、資料の直接検索・権威経由の探索・学術探索を同時に行い、
5つのファイル（ブリーフィング・テーマ概要・参考資料・学びと示唆・代表事例）を出力する。

ユーザー入力は `$ARGUMENTS` で受け取る。

---

## 進捗表示

各フェーズの開始時・完了時に進捗を表示する。

実績データが `output/.stats/session-log.jsonl` に存在する場合は、
直近10回の平均から残り時間を予測する。存在しない場合はハードコード値を使用
（quick: 15分, standard: 30分, deep: 60分）。

表示フォーマット:
```
[Phase N/4] フェーズ名 完了 [████░░░░░░] XX% (残り約YY分)
```

Phase 3 はバッチ完了数で按分:
```
[Phase 3/4] DEEP SUMMARY 進行中 [██████░░░░] 55% -- 2/4バッチ完了 (残り約12分)
```

## Phase 0: INTAKE（入力受付・文脈確認）

### 入力のパース

`$ARGUMENTS` を解析し、トピックと背景を分離する。

- 句点（。）、改行、「背景:」「context:」等の区切りがあれば、前半をトピック、後半を背景とする
- 区切りがなければ全体をトピックとし、背景なしと判定する

### Step 0-1: 文脈の確定

**背景がある場合:**
背景の解釈をユーザーに1-2文で要約して提示する。
曖昧な点がある場合のみ1問で確認する（最大1往復）。

**背景がない場合:**
文脈候補を提示して確認する（ブロッキング）:

トピックから想定される文脈を5つ + フリーテキスト選択肢を生成。
候補生成の方針:
1. **立場軸**で2-3個: 同じトピックでも立場で見え方が変わる
2. **目的軸**で1-2個: 実務応用 vs 学術理解 vs 投資判断 など
3. **スコープ軸**で1個: 広く浅く vs 特定サブトピックに深く

```
以下のどの文脈に近いですか？（複数選択・自由記述OK）

1. [立場軸の候補1]
2. [立場軸の候補2]
3. [立場軸の候補3]
4. [目的軸の候補]
5. [スコープ軸の候補]
6. その他（自由に記述してください）
```

### Step 0-2: 実行モード選択

文脈が確定したら、以下を提示:

```
実行モードを選んでください:

1. お任せ（スタンダードで自動実行。完了まで待つだけ）
2. カスタム（深さ・資料を自分で選択）
```

### お任せモード（1を選択）

- **depth**: standard 固定
- **resource_review**: skip（資料の事前確認なし）
- 即座に Phase 1 を開始する

### カスタムモード（2を選択）

**Step 2-a: 深さ選択**
```
どの程度の深さで調査しますか？

1. クイック（15分目安）: 全体像 + 主要資料5件 + 事例3件
2. スタンダード（30分目安）: 全体像 + 主要資料20件（日本10/海外10）+ 事例5件
3. ディープ（60分目安）: 全体像 + 主要資料30件以上 + 事例8件 + 論文調査を強化
```

**Step 2-b: 資料確認の選択（深さ回答の直後に提示）**
```
資料リストの事前チェックはどうしますか？

A. そのまま進める（AIが選んだランキングで自動進行）
B. チェックしたい（Phase 1 完了後にランキングを見て入れ替え可能）
```

→ Phase 1 を開始する

### Phase 0 の完了条件

以下が確定したら Phase 1 に進む:
- **topic**: トピック（文字列）
- **context**: 背景・課題感（文字列）
- **depth**: quick / standard / deep
- **mode**: omakase / custom
- **resource_review**: skip / check（omakase の場合は常に skip）
- **topic_type**: 実務 / 学術 / 技術（トピックと文脈から自動判定。ユーザーには聞かない）

進捗表示: `[Phase 0/4] INTAKE 完了 [█░░░░░░░░░] 5%`

## Phase 1: SURVEY（並列探索）

3つのsubagentを **同時に** 起動し、並列で探索を行う。

進捗表示: `[Phase 1/4] SURVEY 開始... (予測: 残り約XX分)`

### 作業ディレクトリの作成

トピック名と日付からセッション固有のディレクトリを作成し、中間ファイルの衝突を防ぐ。

```bash
# トピック名を正規化（スペース→ハイフン、特殊文字除去）し、日付と組み合わせる
# 例: "プライシング" + "2026-03-30" → output/catchup-プライシング-2026-03-30/
SESSION_DIR="output/catchup-[正規化トピック名]-[YYYY-MM-DD]"
mkdir -p "$SESSION_DIR/.intermediate"
```

以降すべての中間ファイル・最終出力はこの `$SESSION_DIR` 配下に格納する。
同じトピック・同日に再実行した場合は末尾に連番を付与する（例: `-2`）。

## Phase 2: EVALUATE & RANK（評価・統合）

3つのsubagentが完了したら、中間ファイルを読み込んで統合処理を行う。
このフェーズはメインエージェントが直接実行する。

進捗表示: `[Phase 2/4] EVALUATE & RANK 開始... (予測: 残り約XX分)`

### Step 2-1: 中間ファイルの読み込み

以下の3ファイルを読み込む:
- `$SESSION_DIR/.intermediate/survey-direct.json`
- `$SESSION_DIR/.intermediate/survey-authority.json`
- `$SESSION_DIR/.intermediate/survey-academic.json`

### Step 2-2: マージ・名寄せ

- 3つの結果から全資料を統合する
- 同一資料の重複を排除する（タイトル・URL・著者で照合）
- 複数経路で発見された資料にボーナススコアを付与

### Step 2-3: スコアリング

`./references/evaluation-criteria.md` を参照し、各資料にスコアを付与する。
トピック特性（実務/学術/技術）に応じた重みを使用する。

### Step 2-4: ランキング生成

- 海外資料 上位10件
- 日本語資料 上位10件
- 合計20件（standard の場合。quick=5件、deep=30件以上）

### Step 2-5: 全体像の骨格構築

資料群と各subagentの emerging_themes / field_structure から全体像の骨格を構築する:
- 主要な論点軸と対立構造
- 時系列的な発展経緯
- 現在のトレンドと今後の方向性
- サブトピック間の関係性

骨格を `$SESSION_DIR/.intermediate/overview-skeleton.md` に保存する。

### Step 2-6: ユーザー確認（モードにより分岐）

#### resource_review = skip（お任せ or カスタム+A）の場合

ランキングを**参考表示**し、確認を求めずにそのまま進行する:

```
以下の[N]件で深堀りに進みます。変更があれば今すぐ送ってください。

【海外資料 Top 10】
1. "[タイトル]" - [著者] ★★★★★
2. ...

【日本語資料 Top 10】
1. "[タイトル]" - [著者] ★★★★★
2. ...
```

**ユーザーの回答を待たずに**、資料リストを確定して `$SESSION_DIR/.intermediate/ranked-resources.json` に保存し、Phase 3 を開始する。

**割り込み対応:**
- Phase 3 開始前にユーザーから変更指示があった場合 → 反映して Phase 3 開始
- Phase 3 開始後にユーザーから変更指示があった場合 → 「既に深堀りを開始しています。次回の実行時に反映します」と伝える

#### resource_review = check（カスタム+B）の場合

ランキングをユーザーに提示し、確認を**ブロッキングで待つ**:

```
以下の[N]件を深堀り対象とします。変更はありますか？

【海外資料 Top 10】
1. "[タイトル]" - [著者] ★★★★★
2. ...

【日本語資料 Top 10】
1. "[タイトル]" - [著者] ★★★★★
2. ...

選択肢:
A. この[N]件で進める
B. 件数を変更する（数を指定してください）
C. 特定の資料を入れ替える（番号と代替を指定してください）
D. 追加で調べてほしいトピック・資料がある
```

ユーザーの選択に応じて資料リストを調整し、確定した資料リストを
`$SESSION_DIR/.intermediate/ranked-resources.json` に保存する。

進捗表示: `[Phase 2/4] EVALUATE & RANK 完了 [█████░░░░░] 40%`

## Phase 3: DEEP SUMMARY + EXAMPLES（並列深堀り + 事例調査）

確定した資料リストの深堀りと、代表事例の調査を並列で行う。

進捗表示: `[Phase 3/4] DEEP SUMMARY + EXAMPLES 開始... (予測: 残り約XX分)`

### バッチ分割（資料深堀り）

資料リストを3件ずつのバッチに分割する。
例: 20件の場合 → 7バッチ（3件×6 + 2件×1）

**バッチサイズが5件だとsubagentの出力制約により後半の資料が省略されるリスクがある。
3件を上限とすること。**

バッチはランキング順に分割する（02_references.md での `cat` 結合時にそのまま使えるように）:
- batch-1: 海外 Top 1-3
- batch-2: 海外 Top 4-6
- batch-3: 海外 Top 7-10（端数は4件まで許容）
- batch-4: 日本 Top 1-3
- batch-5: 日本 Top 4-6
- batch-6: 日本 Top 7-10（端数は4件まで許容）

### Subagent起動（資料深堀り）

各バッチについて、agentプロンプトテンプレート `./agents/deep-summary.md` を読み込み、
subagentを起動する。

以下のコンテキストを注入:

> **担当資料リスト:**
>
> [このバッチの資料リスト（JSON）]
>
> **全体像の骨格:**
>
> [$SESSION_DIR/.intermediate/overview-skeleton.md の内容]
>
> 出力フォーマットは `./references/output-format.md` の資料種別ごとのサマリーフォーマットに従ってください。
>
> 出力先: `$SESSION_DIR/.intermediate/summary-batch-[N].md`

モデル: Sonnet

### Subagent起動（事例調査）

agentプロンプトテンプレート `./agents/research-examples.md` を読み込み、subagentを起動する。

以下のコンテキストを注入:

> **Phase 0 の結果:**
>
> - topic: [確定したトピック]
> - context: [確定した背景]
> - depth: [確定した深さ]
> - topic_type: [判定したトピック特性]
>
> **サブトピック:**
>
> [$SESSION_DIR/.intermediate/overview-skeleton.md から抽出したサブトピック一覧]
>
> **全体像の骨格:**
>
> [$SESSION_DIR/.intermediate/overview-skeleton.md の内容]
>
> 出力先: `$SESSION_DIR/.intermediate/examples.md`

モデル: Sonnet

### すべてのsubagentを並列で起動する

資料深堀りの全バッチ + 事例調査の **すべてのAgent toolの呼び出しを1つのメッセージ内で同時に** 行うこと。

進捗表示（バッチ完了ごと）:
```
[Phase 3/4] DEEP SUMMARY + EXAMPLES 進行中 [██████░░░░] 55% -- 2/4バッチ + 事例調査完了
```

進捗表示（全subagent完了後）: `[Phase 3/4] DEEP SUMMARY + EXAMPLES 深堀り完了 [██████░░░░] 65%`

### バリデーション（Phase 3 完了後、Phase 4 開始前）

全バッチファイルの品質を検証し、不足があれば補修する。

#### Step 3-V1: 必須セクションの存在チェック

各バッチファイルに対し、Bashではなく **Grep ツールと Read ツール** で以下を確認する:

1. 各バッチファイルを Grep で `^#### 資料/` を検索し、資料数をカウント
2. 同じファイルを Grep で `ストーリーライン` を検索し、セクション数をカウント
3. 同じファイルを Grep で `主要な主張` を検索し、セクション数をカウント

各バッチで「資料数 = ストーリーライン数 = 主要な主張数」が成立すればOK。
（論文の場合、主要な主張の代わりに品質評価があれば許容する）

**注意: バリデーションに Bash は使わない。Grep/Read ツールを使うこと。**

#### Step 3-V2: 品質チェック

各バッチファイルを読み込み、`./references/summary-requirements.md` の品質要件と照合する。
要件を満たさない資料エントリを特定し、不足内容をリスト化する。

#### Step 3-V3: 補修subagentの起動（最大3回）

不足があった場合、**欠落セクションのみ** を補修する小さなsubagentを起動する。

- **入力**: 不足資料のタイトル・URL・欠落セクション名のリスト
- **出力**: 欠落セクションの内容のみ（Markdown形式）
- **起動上限**: 最大3回まで。3回で解決しない場合は、取得できた範囲で最も充実したものを採用し、不足セクションに「情報不足: WebFetch等で十分な情報を取得できませんでした」と明記する
- **モデル**: Sonnet

補修subagentのプロンプト:

> 以下の資料について、指定されたセクションの内容を生成してください。
> WebSearch/WebFetchで資料の情報を取得し、フォーマットに従って出力してください。
>
> **補修対象:**
> [不足資料と欠落セクションのリスト]
>
> **フォーマット:** `./references/output-format.md` の資料種別ごとのサマリーフォーマットに従う。
> 見出しレベルは `####`（資料名）、`#####`（セクション名）を使用。
>
> 出力先: `$SESSION_DIR/.intermediate/patch-[N].md`

補修完了後、パッチ内容を該当バッチファイルにEditで反映する。

進捗表示: `[Phase 3/4] DEEP SUMMARY + EXAMPLES 完了（バリデーション済み） [████████░░] 75%`

## Phase 4: SYNTHESIZE（全体像の構築）

すべてのPhase 3 subagentが完了したら、最終出力を5ファイルに分けて生成する。
このフェーズはメインエージェントが直接実行する。

進捗表示: `[Phase 4/4] SYNTHESIZE 開始... (予測: 残り約XX分)`

### Step 4-1: 全情報の読み込み

以下を読み込む:
- `$SESSION_DIR/.intermediate/overview-skeleton.md`（Phase 2の全体像骨格）
- `$SESSION_DIR/.intermediate/summary-batch-*.md`（Phase 3の全バッチ結果）
- `$SESSION_DIR/.intermediate/ranked-resources.json`（確定ランキング）
- `$SESSION_DIR/.intermediate/examples.md`（事例調査結果）
- `./references/output-format.md`（出力フォーマット仕様）

### Step 4-2: ① 01_structure.md の生成

`./references/output-format.md` の ① テンプレートに従い、以下を実行:

1. Phase 2の骨格に、Phase 3の各資料サマリーから得た詳細を統合
2. 論点間の関係性を明確化
3. 時系列の発展経緯を整理
4. 各サブトピックの関係性を構造的に記述

### Step 4-2b: 場面別の考慮事項マップ（①のセクション1.7）

1. **観点の抽出**: このテーマに取り組む際に状況を左右する主要な観点（軸）を3-5つ特定する。
   観点は資料群で繰り返し言及される分類軸から抽出する。

2. **場面の特定**: 観点の代表的な組み合わせから、実務上よくある場面を5-10パターン選定する。
   全組み合わせの網羅は不要。Phase 0の文脈に近い場面を優先的に含める。

3. **論点の紐付け**: 各場面で特に重要になる論点と、注意すべき事項を整理する。
   セクション1.3や資料サマリーとの参照を付ける。

### Step 4-2c: 資料横断の深堀り（①のセクション3）

Phase 3の全サマリーを横断的に分析し、以下を生成:

1. **コアメッセージ**: 複数の資料が共通して主張するポイント（3-5点、資料参照付き）
2. **見解の相違**: 資料によって意見が分かれるポイントと背景の違い
3. **時代・地域差**: 海外と日本の視点差、古い資料と新しい資料の変遷
4. **統合サマリー**: 横断的に読んで初めて見える全体像（5-10文）

### Step 4-3: 用語集の生成（①のセクション2）

全体像と資料サマリーから、当該分野の重要用語を抽出する。

各用語について:
- **定義**: 2文以内
- **アナロジー**: 身近な例え
- **なぜ重要か**: 1文

基礎的すぎる用語は除外する（「これを知らないと話についていけない」レベルのみ）。

**①を `$SESSION_DIR/01_structure.md` に保存する。**

### Step 4-4: ② 02_references.md の生成

**重要: 資料サマリーはAIが再生成しない。Phase 3 のバッチファイルを機械的に結合する。**
再生成すると出力量の制約により後半の資料が省略されるため、Bashの`cat`で結合する。

`./references/output-format.md` の ② テンプレートに従い、**2段階で**生成する:

#### Step 4-4-1: ヘッダー部分の生成（Writeで新規作成）

以下をまとめて `$SESSION_DIR/02_references.md` にWriteする:
1. **ファイルヘッダー**: タイトル・相互リンク
2. **推奨読書順序**: 最初に読むべき3冊/記事（理由と所要時間目安付き）+ 論点別の推奨資料
3. **重要資料ランキング**: Phase 2の確定ランキングをテーブル形式で（海外 Top N + 日本語 Top N）
4. **セクション3の見出し**: `## 3. 各資料サマリー` と `### 海外資料` の見出しのみ

**ヘッダー末尾は改行2つで終わること（後続のcat結合のため）。**

#### Step 4-4-2: バッチファイルの機械的結合（Bashで実行）

Phase 3 のバッチファイルを `cat` でそのまま結合する。
**AIによる転記・再生成は一切行わない。**

deep-summary.md のフォーマット指定により、バッチファイルは
既に 02_references.md 用の見出しレベル（`####`/`#####`）で出力されている。

**Bash ではなく Read + Edit ツールで結合すること。**
`SESSION_DIR=... && for ...` のようなBashチェーンは権限確認が発生するため使わない。

手順:
1. 各バッチファイルを **Read ツール** で読み込む
2. 読み込んだ内容を以下の順序で **Edit ツール** を使って `$SESSION_DIR/02_references.md` の末尾に追記する:
   - `### 海外資料` 見出し
   - batch-1, batch-2, batch-3 の内容（海外資料）
   - `---` 区切り
   - `### 日本語資料` 見出し
   - batch-4, batch-5, batch-6 の内容（日本語資料）

**注意: バッチ分割は Phase 3 でランキング順に行うこと。**
海外 Top 1-3 → batch-1、海外 Top 4-6 → batch-2、海外 Top 7-10 → batch-3、
日本 Top 1-3 → batch-4、日本 Top 4-6 → batch-5、日本 Top 7-10 → batch-6。
バッチ番号は depth に応じて増減する（quick=少、deep=多）。

#### Step 4-4-3: 検証

結合完了後、**Grep ツール** で以下を確認する:

1. `$SESSION_DIR/02_references.md` を Grep で `ストーリーライン` を検索し、カウント
2. 同ファイルを Grep で `主要な主張` を検索し、カウント

- ストーリーラインが資料数と一致すること
- 主要な主張が資料数と概ね一致すること（論文は品質評価で代替可）
- 不一致の場合はバッチファイル側の生成不備を疑い、該当バッチのsubagentの出力を確認する

`./references/output-format.md` の ③ テンプレートに従い、
`$SESSION_DIR/.intermediate/examples.md` の事例調査結果を整形して出力する。

**③を `$SESSION_DIR/03_cases.md` に保存する。**

### Step 4-5: ④ 04_insights.md の生成

#### Step 4-5a: 全体的な学びの整理

資料群と事例群の両方から得られる学びを整理:
1. **テーマの本質**: 個別論点を超えたメタレベルの理解（3-5文）
2. **よくある誤解と実態**: 初学者が陥りがちな誤解と資料・事例が示す実態の対比
3. **成功と失敗のパターン**: 資料と事例の両方から抽出した共通パターン（5-10点）

#### Step 4-5b: Gap check + 追加調査

全体像に明らかな抜けがないか自己検証する:
- 資料群で言及されていたが全体像に反映されていない重要論点はないか
- 特定の立場・視点が偏っていないか
- 時系列に大きな空白がないか

**発見された残論点について、WebSearch/WebFetch で追加調査を実施する。**
- 解決できた論点 → 「追加調査で解決した論点」テーブルに記載
- 解決できなかった論点 → 「未解決の論点」テーブルに記載（推奨する調査方法を添えて）

#### Step 4-5c: ユーザー文脈への示唆

Phase 0で確定した文脈に基づき:
- 課題に特に関連する論点を優先度順に整理
- 推奨するアクション（3-5点）
- 判断に迷いやすいポイントと判断の指針

**④を `$SESSION_DIR/04_insights.md` に保存する。**

### Step 4-6: ブリーフィング [トピック]_[YYYYMMDD].md の生成

**①〜④を書き終えた後に、最後に生成する。**

①〜④の各ファイルの内容を参照し、エッセンスを凝縮した概要を生成する。
`./references/output-format.md` のブリーフィングテンプレートに従う。

ファイル名はトピック名と日付から生成する（例: `ファイナンス_20260330.md`）。
単体で他者にシェアできることを意識する。

5-10分で読める分量を目安とし、各セクションは①〜④の該当箇所から抜粋・圧縮する:
- テーマ概要 ← ①の1.1〜1.2を圧縮
- 主要な論点 ← ①の1.3から最重要3-5点
- 全体の概念構造 ← ①の1.6 サブトピックの関係性をそのまま記載
- 場面別の要注意ポイント ← ①の1.7からユーザー文脈に近い2-3場面
- 注目事例 ← ③から最も参考になる1-2事例を抜粋
- 重要資料 Top 5 ← ②のランキングから上位5件
- 主要な学び ← ④の全体的な学びから最重要3-5点
- 今回の文脈への示唆 ← ④の文脈への示唆を1段落に圧縮

**ブリーフィングを `$SESSION_DIR/[トピック]_[YYYYMMDD].md` に保存する。**

進捗表示: `[Phase 4/4] SYNTHESIZE 完了 [██████████] 100%`

### Step 4-7: 実績記録 + 完了報告

#### 実績の記録

セッションの所要時間を `output/.stats/session-log.jsonl` に追記する。

```bash
mkdir -p output/.stats
```

以下のJSON形式で1行追記:

```json
{
  "date": "YYYY-MM-DD",
  "topic": "[トピック]",
  "depth": "quick|standard|deep",
  "topic_type": "実務|学術|技術",
  "resource_count": 20,
  "example_count": 5,
  "timing_ms": {
    "phase0": 0,
    "phase1_survey_direct": 0,
    "phase1_survey_authority": 0,
    "phase1_survey_academic": 0,
    "phase1_wall": 0,
    "phase2": 0,
    "phase3_batches": [0, 0, 0, 0],
    "phase3_examples": 0,
    "phase3_wall": 0,
    "phase4": 0,
    "total_wall": 0
  }
}
```

各フェーズの `duration_ms` はsubagent完了通知から取得する。
メインエージェントが直接実行するフェーズは、開始・終了時刻の差分から算出する。
